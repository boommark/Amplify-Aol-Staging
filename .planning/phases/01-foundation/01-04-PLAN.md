  <action>
1. Create `lib/prompts/extract-n8n.ts` — a Node.js script (runnable with `npx tsx`) that:
   - Reads all JSON files from `../n8n amplify scripts/` directory
   - Parses each n8n workflow JSON structure
   - n8n workflow nodes have a `type` field. Look for nodes with AI/LLM-related types:
     - `@n8n/n8n-nodes-langchain.lmChatOpenAi` or similar LLM nodes
     - `n8n-nodes-base.httpRequest` nodes that call AI APIs
     - `@n8n/n8n-nodes-langchain.agent` nodes
     - Any node with parameters containing `prompt`, `systemMessage`, `text`, or `messages`
   - For each prompt found, extract:
     - The prompt text (from `parameters.prompt`, `parameters.text`, `parameters.systemMessage`, `parameters.messages`, or similar fields)
     - The model/provider info if available (from node parameters or connected LLM nodes)
     - The workflow name (from the filename) for context
   - Categorize each prompt with a key following `domain.task` convention:
     - `research.regional` — regional research/market research prompts
     - `research.translated` — translated research prompts
     - `copy.email` — email copy generation
     - `copy.whatsapp` — WhatsApp message generation
     - `copy.instagram` — Instagram post copy
     - `copy.facebook` — Facebook post copy
     - `copy.intro-talk` — intro talk/presentation content
     - `wisdom.quotes` — Gurudev quote selection/curation
     - `wisdom.questions` — questions for Ask Gurudev API
     - `image.ad-creative` — ad image prompt generation
     - `image.quote` — quote image generation
     - `ads.copy` — ad copy (headline, subheadline, CTA)
     - `ads.national` — national ad campaign prompts
     - `translation.content` — content translation prompts
   - Assign `model_override` based on the provider found in the workflow (e.g., `openai/gpt-4o`, `google/gemini-2.0-flash`, `anthropic/claude-sonnet-4-5`)
   - Output a SQL file to `supabase/seed/prompts.sql`
   - Also log a summary to stdout: count of prompts found per workflow, total prompts, keys assigned

2. The generated `supabase/seed/prompts.sql` must:
   - Begin with a comment header documenting extraction date and source
   - Use proper SQL escaping for prompt text (escape single quotes by doubling them)
   - Insert each prompt as: `INSERT INTO prompts (key, version, template, model_override, description, is_active, created_at) VALUES (...)`
   - All inserts use `version = 1` and `is_active = true`
   - Include `description` field noting the source workflow filename
   - Use `ON CONFLICT DO NOTHING` or a conditional insert to be idempotent
   - End with a count verification query: `SELECT key, version FROM prompts WHERE version = 1 ORDER BY key;`

3. Run the extraction script:
   ```bash
   npx tsx lib/prompts/extract-n8n.ts
   ```
   Review the output summary and verify the seed SQL looks correct.

**Important notes:**
- n8n JSON structure: top-level has `nodes` array. Each node has `type`, `name`, `parameters`. Prompt text is typically in `parameters.prompt`, `parameters.text`, or nested in `parameters.messages[].content`.
- Some prompts may use n8n expression syntax like `{{ $json.field }}` — preserve these as-is in the template (they document the expected variables for the prompt).
- If a prompt references n8n-specific fields, add a SQL comment noting the variable mapping.
- Deduplicate prompts that appear across multiple workflows (same text = same prompt, keep one with the best key).
