# Plan 03-06: Workshop URL Auto-Parser

<objective>
Enable users to paste a workshop URL and have Amplify automatically extract event details (title, date, location, description, price, event type) and proceed directly into the content pipeline — no manual description needed.
</objective>

## Task 1: URL parser module and API route

**Files:** `lib/pipeline/url-parser.ts`, `app/api/pipeline/parse-url/route.ts`

1. Create `lib/pipeline/url-parser.ts`:
   - `parseWorkshopUrl(url: string)` → fetches the page HTML, extracts structured event data
   - Use `fetch()` to get the page HTML with a browser-like User-Agent
   - Extract from HTML using regex/string matching (no heavy DOM parser dependency):
     - `<title>` or `og:title` → event title
     - `og:description` or meta description → event description
     - Look for date patterns, location/address, price info in the page content
   - For Art of Living URLs (artofliving.org): parse their specific event page structure
   - For generic URLs: use Claude Haiku to extract structured data from the page text (truncated to 4000 chars)
   - Return type: `{ title, date, location, region, eventType, description, price, source }`
   - Graceful error handling: if scraping fails, return partial data with what was found

2. Create `app/api/pipeline/parse-url/route.ts`:
   - POST endpoint with auth guard
   - Accepts `{ url, campaignId }`
   - Calls `parseWorkshopUrl(url)`
   - Updates campaign record with extracted region, event_type, title
   - Returns parsed data
   - `maxDuration = 60`

## Task 2: Integrate URL detection into pipeline orchestrator and chat flow

**Files:** `lib/pipeline/orchestrator.ts`, `app/api/chat/route.ts`, `hooks/usePipelineChat.ts`

1. Update `lib/pipeline/orchestrator.ts`:
   - Add new intent type: `{ type: 'url_parse'; url: string }`
   - In `detectPipelineIntent`: add URL detection before AI classification
   - Simple regex: if message contains a URL (https?://...), extract it and return `url_parse` intent
   - URL detection should be the FIRST check (before action chip matching)

2. Update `app/api/chat/route.ts`:
   - Add handler for `intent.type === 'url_parse'`:
     - Call `parseWorkshopUrl(url)`
     - Update campaign with parsed data
     - Auto-trigger research pipeline with parsed region/eventType
     - Return progressive SSE stream (same as research) so user sees parsing → research flow seamlessly
   - Import `parseWorkshopUrl` from `lib/pipeline/url-parser`

3. Update `hooks/usePipelineChat.ts`:
   - Add `url_parsing` action handler in `handlePipelineResponse`
   - Show a "Parsing workshop details..." state while URL is being processed
   - Once parsed, transition smoothly into research stage
